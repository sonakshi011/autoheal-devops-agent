# API Guide & Reference

The **AutoHeal DevOps Agent** exposes a lightweight, secure, and observable REST API built with FastAPI. It enforces strict route versioning under the `/api/v1` prefix and returns standardized JSON response envelopes.

---

## 1. Unified Response Envelopes

To prevent schema complexity and ensure frontend typing stability across all endpoints, AutoHeal wraps all FastAPI responses in standardized envelopes:

### Success Envelope Format
```json
{
  "success": true,
  "data": {},
  "timestamp": "2026-05-17T18:00:00.000000+00:00"
}
```

### Error Envelope Format (Centralized Exceptions)
All errors—including Starlette HTTPExceptions, validation errors, and runtime failures—are caught by global handlers in `app/main.py` and wrapped into:
```json
{
  "success": false,
  "error": "Detailed error message or validation explanation.",
  "timestamp": "2026-05-17T18:00:00.000000+00:00"
}
```

## 2. Global Unversioned Routes

### `GET /`
Basic root endpoint to check if the FastAPI instance is listening.
- **Success Response**:
  ```json
  {
    "message": "Welcome to AutoHeal DevOps Agent API"
  }
  ```

### `GET /health/`
Detailed status check used by Docker Compose and load-balancer health engines.
- **Success Response**:
  ```json
  {
    "status": "healthy",
    "version": "0.1.0"
  }
  ```

### `GET /metrics`
Exposes the Prometheus ASGI metrics register (e.g. latency histograms, request counters).
- **Format**: OpenMetrics / Prometheus plaintext format.

## 3. Versioned Operational Routes (`/api/v1`)

### `GET /api/v1/pipelines/runs`
Fetches the latest 10 GitHub Actions workflow runs dynamically using PyGithub (or falls back to dev mock data if tokens are not configured).
- **Success Response**:
  ```json
  {
    "success": true,
    "data": {
      "runs": [
        {
          "id": 5519283748,
          "name": "Security Scan & Lint",
          "status": "completed",
          "conclusion": "failure",
          "event": "push",
          "html_url": "https://github.com/owner/repo/actions/runs/5519283748",
          "created_at": "2026-05-17T10:00:00Z"
        }
      ],
      "is_mock": false
    },
    "timestamp": "2026-05-17T18:00:00.123456+00:00"
  }
  ```

### `GET /api/v1/ai/latest-diagnosis`
Retrieves the latest AI diagnosis JSON report generated by the Gemini diagnostic engine.
- **Synchronization Logic**: This route is dynamically linked to the latest GitHub workflow run:
  - If the latest run succeeded (`conclusion == "success"`), the endpoint automatically clears stale state and returns a **`404 Not Found`** envelope:
    ```json
    {
      "success": false,
      "error": "No active failures detected. The latest workflow run succeeded.",
      "timestamp": "2026-05-17T18:00:02.111222+00:00"
    }
    ```
  - If the latest run failed, it reads and serves the active `reports/ai_diagnosis.json` report.
- **Success Response (Latest Run Failed)**:
  ```json
  {
    "success": true,
    "data": {
      "root_cause": "The build failed due to an unhandled import statement in main.py.",
      "severity": "critical",
      "remediation_steps": [
        "Check that all packages in requirements.txt are installed.",
        "Verify imports match python 3.12 syntax."
      ],
      "code_suggestion": "import os\n# Added missing module import",
      "confidence_score": 95,
      "model_used": "gemini-2.5-flash",
      "diagnosis_timestamp": "2026-05-17T10:15:00Z"
    },
    "timestamp": "2026-05-17T18:00:00.222333+00:00"
  }
  ```

### `GET /api/v1/scans/trivy`
Reads and validates the Trivy vulnerability scanner filesystem findings from the reports directory.
- **Success Response**:

  {
    "success": true,
    "data": {

## 4. Request Correlation & Tracing

All versioned routes use an `X-Request-ID` correlation middleware:
- **Preserved Tracing**: If a request comes in with an `X-Request-ID` header, the backend preserves it.
- **Automatic Generation**: If the header is missing, the backend generates a random `UUIDv4`.
- **Trace Propagation**: The ID is returned in the response headers as `X-Request-ID`, and injected into every structured log line output to `stdout`. This enables instant log correlation inside Loki.

## 5. Structured JSON Logging Format

Uvicorn and FastAPI console outputs are structured into single-line JSON statements for reliable Promtail scraping:

```json
{
  "timestamp": "2026-05-17T18:12:04.222Z",
  "level": "INFO",
  "logger_name": "autoheal.api",
  "message": "http_request",
  "service_name": "autoheal-devops-agent",
  "environment": "production",
  "request_id": "57ad54bf-7793-43c0-9ddf-f13e7af51ecf",
  "method": "GET",
  "path": "/api/v1/ai/latest-diagnosis",
  "status_code": 404,
  "latency_ms": 2.17
}
```
